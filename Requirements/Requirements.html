Project Requirements
Project Summary

This semester we are interested in developing projects based on standards and modularity. The course theme this semester is a STEM-themed display that demonstrates a single scientific/engineering concept in an interactive, web-enabled, physical device. This could include displays such as:

  • The Bernoulli Effect
  • Self-balancing Inverted Pendulum
  • A line following robot
  • ...
    • The intended user is, thus, K-12 students interested in learning about science, technology, engineering, or math.

    Project Requirements

    Our team's project requirements come from two different sources.

  • Course-Imposed requirements (Requirements from the class)
  • Hardware-Imposed requirements (Requirements due to hardware selection)

    • Course-Imposed Restrictions

    The project have specified requirements by the teaching team, they are as follows.

  • Microcontroller: Must use an Espressif ESP32 or a Microchip PIC16F/PIC18F
  • Power: Must use a 3.3V switching regulator, powered from a 120v -> 9v power supply
  • In-Circuit Programmer: Either a USB for the ESP32, or all supporting hardware for ICSP for PIC microcontrollers
  • Functionalities: Either features a sensor, actuator, human machine interface, or bidirectional internet communication.
  • Communication: Systems must intercommunicate utilizing UART

    • Hardware-Imposed Restrictions

    The project have specified requirements due to the hardware we selected, they are as follows

  • Internet-Connectivity Our device must be able to interface with our website, giving live-data readouts
  • Power: We additionally require a 24v power supply to drive both the stepper motors and the primary fan
  • Communication: We will use a global send/recieve system, where addresses and cyclical sending minimize communication traffic
